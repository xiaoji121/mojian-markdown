# 墨笺 Markdown 宣传方案与发布文案

## 核心传播策略

主定位：不是“又一个 Markdown 编辑器”，而是“为认真阅读而做的开源 Markdown 工作台”。

传播顺序：先用“不是写完，是读懂”制造认知差异，再展示阅读闭环和真实界面，最后用开源、MIT、无需安装的低门槛促成体验。

优先人群：经常阅读产品方案、技术文档、课程材料和论文笔记的人；习惯 Markdown 的开发者、产品经理、研究者与终身学习者。

内容支柱：

1. 痛点型：为什么读过很多，却很少真正沉淀？
2. 产品型：网页剪藏 → 阅读 → 批注 → AI 追问 → 归档回看。
3. 场景型：读 PRD、接口文档、论文笔记、课程材料。
4. 开源型：真实构建过程、设计取舍、版本迭代和用户反馈。

建议节奏：首周连续发布 3 篇（产品发布、功能实拍、使用场景），之后每周 1 篇案例或版本日志。所有帖子只保留一个行动目标：体验产品或收藏 GitHub。

### 正文

我一直觉得，很多 Markdown 工具把“写完”做得很好，却很少照顾“读懂”。

读一篇产品方案或技术文档时，我真正需要的是：原文和预览能快速对应；看到重点可以直接划线、写下疑问；问过 AI 的答案不要转身就丢；下次打开还能从上次停下的地方继续。

所以我做了「墨笺 Markdown」。

它把这些动作放进了同一个阅读工作台：

- Markdown 直接编辑，预览实时同步，预览区也能修改
- 原文与预览双击双向定位，长文不用来回找
- 马克笔、波浪线、直线和想法批注，集中收进批注面板
- 浏览器本地缓存，支持导入、导出和继续阅读
- 本地完整版支持选中问 AI，并把问答归档到当前文档
- 配套浏览器插件可把网页正文、整页或选中内容转成 Markdown

线上体验版打开网页就能用；完整版本地运行，MIT 协议开源。

我想做的不是另一个写作工具，而是一个能让阅读留下痕迹的地方。

项目地址：github.com/xiaoji121/mojian-markdown

如果你也经常读 PRD、接口文档、论文笔记或课程材料，欢迎试试，也很想听听你最希望补上的阅读功能。
